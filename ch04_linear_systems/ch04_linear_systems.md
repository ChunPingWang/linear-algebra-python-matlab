# 第 4 章：線性方程組與高斯消去法

## 學習目標

讀完本章後，你應該能夠：

- 將一組線性方程式改寫成矩陣形式 `Ax = b`，並理解各部分的意義
- 建立增廣矩陣 `[A|b]`，並用它來代表整個方程組
- 熟悉高斯消去法的三種基本列運算，並能手算化簡一個 3 元一次方程組
- 分辨列梯形式 (Row Echelon Form, REF) 與簡化列梯形式 (Reduced Row Echelon Form, RREF) 的差異
- 判斷一個線性方程組屬於「唯一解」「無限多解」還是「無解」，並理解背後的幾何意義
- 使用 Python（`numpy`）與 MATLAB 求解線性方程組，並手刻一個高斯消去函式來加深理解

## 概念說明

### 1. 線性方程組的矩陣表示 Ax = b

考慮一組線性方程式：

```
2x +  y -  z =   8
-3x -  y + 2z = -11
-2x +  y + 2z =  -3
```

我們可以把所有係數整理成一個矩陣 `A`、把未知數整理成一個向量 `x`、把等號右邊的常數整理成一個向量 `b`：

```
A = [ 2   1  -1]      x = [x]      b = [  8]
    [-3  -1   2]          [y]          [-11]
    [-2   1   2]          [z]          [ -3]
```

於是整個方程組可以精簡地寫成一個矩陣方程式：

```
A x = b
```

這個寫法的好處是：無論方程式有幾個未知數，我們都可以用同一套矩陣運算方法（高斯消去法）來求解，不需要針對「2 元」「3 元」「10 元」方程組分別設計不同解法。

### 2. 增廣矩陣 [A|b]

在做高斯消去的過程中，我們其實只在意「係數」與「常數」，未知數的名稱（x, y, z）並不重要。因此我們把 `A` 和 `b` 併在一起，中間用一條虛線隔開，稱為**增廣矩陣 (augmented matrix)**：

```
[A|b] = [ 2   1  -1 |   8]
        [-3  -1   2 | -11]
        [-2   1   2 |  -3]
```

之後所有的列運算都直接對這個增廣矩陣進行，最後再把結果「翻譯」回方程式即可。

### 3. 高斯消去法的三種基本列運算

高斯消去法 (Gaussian elimination) 的核心精神是：透過一系列不會改變解集合的操作，把增廣矩陣化簡成容易讀出答案的形式。這些操作稱為**基本列運算 (elementary row operations)**，一共有三種：

| 運算 | 記法 | 說明 |
|---|---|---|
| 交換兩列 | `R_i <-> R_j` | 兩條方程式的順序對調，不影響解 |
| 某列乘上非零常數 | `R_i -> k * R_i`（`k ≠ 0`） | 方程式兩邊同乘一個非零常數，不影響解 |
| 某列加上另一列的倍數 | `R_i -> R_i + k * R_j` | 用一條方程式的倍數去消去另一條方程式中的某個未知數 |

**為什麼這三種操作不會改變解？** 因為它們都只是把「同一組方程式」用不同的方式表達出來——本質上仍是原本那些限制條件的線性組合，因此原本的解仍然滿足新方程組，新方程組的解也仍滿足原方程組。

### 4. 列梯形式 (REF) 與簡化列梯形式 (RREF)

高斯消去法的目標，是把增廣矩陣化簡成以下兩種標準形式之一：

- **列梯形式 (Row Echelon Form, REF)**：
  - 每一列的第一個非零元素（稱為**主元 pivot**）都在上一列主元的右邊
  - 全為 0 的列（如果有）都在矩陣最下方

- **簡化列梯形式 (Reduced Row Echelon Form, RREF)**：
  - 滿足 REF 的所有條件
  - 每個主元都是 1
  - 每個主元所在的欄位，除了主元本身外，其餘元素全部為 0

RREF 是「化簡到極致」的形式：一旦化到 RREF，通常可以直接讀出每個變數的解（唯一解的情況），或直接看出哪些是自由變數（無限多解的情況）。

### 5. 手算範例：完整高斯消去過程

我們以第 1 節的方程組為例，示範完整的手算過程：

```
2x +  y -  z =   8
-3x -  y + 2z = -11
-2x +  y + 2z =  -3
```

**初始增廣矩陣：**

```
[ 2   1  -1 |   8]
[-3  -1   2 | -11]
[-2   1   2 |  -3]
```

**步驟 1：`R2 -> R2 + 1.5 R1`**（消去 R2 第一欄的 `-3`，因為 `-3 + 1.5 × 2 = 0`）

```
[ 2    1   -1  |  8]
[ 0   0.5  0.5 |  1]
[-2    1    2  | -3]
```

**步驟 2：`R3 -> R3 + 1.0 R1`**（消去 R3 第一欄的 `-2`，因為 `-2 + 1 × 2 = 0`）

```
[ 2    1   -1  |  8]
[ 0   0.5  0.5 |  1]
[ 0    2    1  |  5]
```

**步驟 3：`R3 -> R3 - 4 R2`**（消去 R3 第二欄的 `2`，因為 `2 - 4 × 0.5 = 0`）

```
[ 2    1    -1  |  8]
[ 0   0.5   0.5 |  1]
[ 0    0    -1  |  1]
```

此時矩陣已經是**列梯形式 (REF)**：每列主元依序右移，且下方沒有全零列夾在中間。

**步驟 4：把每個主元都化為 1**（`R1 -> R1/2`，`R2 -> R2/0.5`，`R3 -> R3/(-1)`）

```
[ 1   0.5  -0.5 |  4]
[ 0    1    1   |  2]
[ 0    0    1   | -1]
```

**步驟 5：回代消去（back-substitution），把每個主元欄位的其他元素都消成 0**

先用 R2 消去 R1 的第二欄（`R1 -> R1 - 0.5 R2`）：

```
[ 1   0   -1  |  3]
[ 0   1    1  |  2]
[ 0   0    1  | -1]
```

再用 R3 消去 R1、R2 的第三欄（`R1 -> R1 + 1 R3`，`R2 -> R2 - 1 R3`）：

```
[ 1   0   0 |  2]
[ 0   1   0 |  3]
[ 0   0   1 | -1]
```

這就是**簡化列梯形式 (RREF)**。從最右邊一欄可以直接讀出解：

```
x = 2,  y = 3,  z = -1
```

**驗證**：代回原方程組第一式，`2(2) + 3 - (-1) = 4 + 3 + 1 = 8`，成立。讀者可自行驗證另外兩式。

### 6. 解的三種情況

一個線性方程組 `Ax = b` 化簡到 REF／RREF 之後，恰好會落入以下三種情況之一：

#### (a) 唯一解 (Unique Solution)

當方程式數量足夠、且每個未知數都對應到一個主元（沒有自由變數），也沒有出現矛盾時，方程組只有一組解。

範例：

```
 x +  y = 3
2x -  y = 0
```

化簡後 RREF 為 `[1 0 | 1; 0 1 | 2]`，解為 `x = 1, y = 2`（幾何意義：兩條直線恰好交於一點）。

#### (b) 無限多解 (Infinite Solutions，含自由變數)

當某些欄位沒有主元（該未知數沒有被「鎖定」），這些未知數稱為**自由變數 (free variable)**，可以任意取值，因此有無限多組解。

範例：

```
 x +  y +  z = 6
2x + 2y + 2z = 12     （第二式其實是第一式的 2 倍，沒有提供新資訊）
```

化簡後 RREF 為 `[1 1 1 | 6; 0 0 0 | 0]`，只有 1 個主元卻有 3 個未知數，因此有 `3 - 1 = 2` 個自由變數。解集合是三維空間中的一個平面，含有無限多組解。

#### (c) 無解 (No Solution，矛盾方程式)

當化簡過程中出現形如 `[0 0 ... 0 | c]` 且 `c ≠ 0` 的一列時，代表「`0 = c`」這種不可能成立的矛盾，此時方程組無解。

範例：

```
x + y = 2
x + y = 5     （與第一式矛盾：x+y 不可能同時等於 2 與 5）
```

化簡後會出現 `[0 0 | 3]` 這種矛盾列，因此無解（幾何意義：兩條平行但不重合的直線，永遠不會相交）。

## Python 實作

Python 使用 `numpy` 的 `np.linalg.solve` 可以快速求解唯一解的方程組，但為了理解演算法本身，我們也手刻了一個 `gaussian_elimination(A, b)` 函式，將增廣矩陣化為 RREF 並判斷解的類型。以下為核心程式碼片段（完整內容見 `ch04_linear_systems.py`）：

```python
import numpy as np

def gaussian_elimination(A, b, verbose=False):
    """以高斯消去法求解 Ax = b，回傳 RREF、解的類型、與解向量。"""
    A = np.array(A, dtype=float)
    b = np.array(b, dtype=float).reshape(-1, 1)
    M = np.hstack([A, b])
    n_rows, n_cols = M.shape
    n_vars = n_cols - 1

    pivot_row = 0
    pivot_cols = []

    for col in range(n_vars):
        if pivot_row >= n_rows:
            break
        # 部分主元法：選該欄絕對值最大的列，增加數值穩定性
        max_row = pivot_row + np.argmax(np.abs(M[pivot_row:, col]))
        if np.isclose(M[max_row, col], 0.0):
            continue
        if max_row != pivot_row:
            M[[pivot_row, max_row]] = M[[max_row, pivot_row]]
        M[pivot_row] = M[pivot_row] / M[pivot_row, col]
        for r in range(n_rows):
            if r != pivot_row and not np.isclose(M[r, col], 0.0):
                M[r] = M[r] - M[r, col] * M[pivot_row]
        pivot_cols.append(col)
        pivot_row += 1

    rank_A = len(pivot_cols)

    # 判斷是否有矛盾列 [0...0 | c], c != 0
    inconsistent = any(
        np.allclose(M[r, :n_vars], 0.0) and not np.isclose(M[r, -1], 0.0)
        for r in range(n_rows)
    )

    if inconsistent:
        return M, "none", None
    elif rank_A < n_vars:
        return M, "infinite", None
    else:
        x = np.zeros(n_vars)
        for i, col in enumerate(pivot_cols):
            x[col] = M[i, -1]
        return M, "unique", x
```

用 `np.linalg.solve` 交叉驗證：

```python
A = np.array([[2, 1, -1], [-3, -1, 2], [-2, 1, 2]], dtype=float)
b = np.array([8, -11, -3], dtype=float)

_, sol_type, x_manual = gaussian_elimination(A, b)
x_numpy = np.linalg.solve(A, b)

print(np.allclose(x_manual, x_numpy))  # True
```

實際執行結果：手刻函式與 `np.linalg.solve` 皆得到 `x = [2, 3, -1]`，兩者一致（詳見 `.py` 檔案執行輸出）。

完整程式（含三種解的情況範例）請見同資料夾中的 [`ch04_linear_systems.py`](ch04_linear_systems.py)，或互動版 [`ch04_linear_systems.ipynb`](ch04_linear_systems.ipynb)。

## MATLAB 實作

MATLAB 求解線性方程組最道地的方式是使用**左除運算子 `\`**：

```matlab
A = [2 1 -1; -3 -1 2; -2 1 2];
b = [8; -11; -3];

x = A \ b;   % 求解 Ax = b，等同於 x = inv(A) * b，但數值上更穩定、更快
disp(x)
```

MATLAB 也內建 `rref()` 函式，可以直接把增廣矩陣化為簡化列梯形式：

```matlab
Aug = [A, b];
R = rref(Aug);
disp(R)   % 最後一欄即為解
```

`rref()` 特別適合用來觀察「無限多解」或「無解」的情況，因為它會清楚地把矛盾列或全零列呈現出來。完整程式碼請見 [`ch04_linear_systems.m`](ch04_linear_systems.m)。

> 注意：本章 `.m` 檔案已用 GNU Octave 10.2 實際執行驗證通過，輸出數值與本章 Python 版本一致；尚未在正式 MATLAB 環境執行，但語法皆為標準 MATLAB 語法，建議你仍自行在 MATLAB 中重新執行一次確認。

## 重點整理

- 任何線性方程組都可以寫成矩陣形式 `Ax = b`，其中 `A` 是係數矩陣、`x` 是未知數向量、`b` 是常數向量。
- 增廣矩陣 `[A|b]` 把係數與常數合併，是高斯消去法操作的對象。
- 高斯消去法只使用三種基本列運算（交換兩列、某列乘上非零常數、某列加上另一列的倍數），這些運算都不改變方程組的解。
- 列梯形式 (REF) 的主元階梯狀排列；簡化列梯形式 (RREF) 更進一步讓每個主元為 1，且該欄其餘元素皆為 0，可直接讀出解。
- 解只有三種可能：唯一解（每個變數都有主元）、無限多解（存在自由變數）、無解（化簡後出現矛盾列 `[0...0|c]`，`c≠0`）。
- Python 可用 `np.linalg.solve` 快速求解，也可手刻高斯消去函式加深理解；MATLAB 可用左除 `A\b` 或 `rref()` 函式。

## 練習題

1. 將下列方程組寫成矩陣形式 `Ax = b`，並寫出對應的增廣矩陣 `[A|b]`：
   ```
   3x - y = 7
   x + 2y = 0
   ```
   **提示**：`A = [3 -1; 1 2]`，`b = [7; 0]`，解為 `x = 2, y = -1`。

2. 手算化簡以下增廣矩陣至 RREF，並判斷解的類型：
   ```
   [1  2 | 5]
   [2  4 | 10]
   ```
   **提示**：第二列是第一列的 2 倍，化簡後會得到一整列全零，屬於「無限多解」（`y` 為自由變數，`x = 5 - 2y`）。

3. 判斷下列方程組的解屬於「唯一解」「無限多解」還是「無解」，並說明理由：
   ```
   x + y = 4
   x + y = 9
   ```
   **提示**：兩式左邊相同但右邊不同，化簡後會出現 `[0 0 | 5]` 的矛盾列，屬於「無解」。

4. 用高斯消去法手算求解：
   ```
    x +  y +  z = 6
   2x -  y +  z = 3
    x + 2y - z  = 2
   ```
   **提示**：先用 `R2 -> R2 - 2R1`、`R3 -> R3 - R1` 消去第一欄，再繼續消去第二欄；正確解為 `x = 1, y = 2, z = 3`。
